「.NETFramework 1.1とはなんぞや？」（ドットネットフレームワーク）
深く知る必要はないけれど、AutoCadの様にインストールを迫られる場合があるので概念くらいは知っておいた方が良いかもね？
Microsoft製品も例外ではなく、Office2007からは「.NET Framework」を先にインストールしないと製品のインストールは出来ません。
大抵の場合は製品のインストーラーが「.NET Framework」のインストールの有無とバージョンの確認を行い、インストールされていなければ（Microsoftのダウンロードセンターからダウンロードして）インストールします。

この行為を行うのがAutoCadの「Setup.exe」ですが、おそらくWindowsXPとWindows8で互換性のない方法で「.NET Framework」のインストーラーを起動しようとしてエラーなったと予想されます。
AutoCadの実質的なインストーラーは、実は「setup.exe」ではなくて
「CD-ROM\\AutoCAD2006 (D)\Bin\acadFeui」の中にある「acad.msi」がインストーラーです。
ですので、以下の手順でインストールできます。
１．NET Framework Runtime 1.1 を Microsoftダウインロードセンターからダウンロードして手動でインストールする。
２．NET Framework Runtime 1.1 Language Pack をMicrosoftダウインロードセンターからダウンロードして手動でインストールする。
３．acad.msi を起動して AutoCad をインストールする。
ちなみに「.msi」拡張子は「Microsoft Windows Installer」の略です。

NET Framework Runtime 1.1 は Windows8.1 で正常に動作するので、AutoCad 2006 も動作すると思います。
但し、T家に譲ったPCはメインメモリがPC-1333の１GB×２＝２GBしかありません。
６４bitOSはメインメモリが４GB以上でないと進化を発揮しません。
逆に２GB程度だとWindowsカーネルのメモリ利用率が高くなり、ユーザーで利用できるメモリが少なくなって逆に遅く感じてしまうかもしれません。
最悪、ヒープ領域が不足してAutoCadが異常終了するかもしれません。
出来れば重い図面を開いて操作感や図面を修正するなどして、正常に動作するかどうかを十分に検証する必要があります。
これはVectorWorksにおいても同じです。
補足しますと「Windows7の方が互換性が高い、動いた、、安心」と言うのはWindows７の32bit版OSの事であり、
Windows７であっても６４bit版OSを使う場合はWindows8（若しくは8.1）と同じ気遣いが必要でしょう。
